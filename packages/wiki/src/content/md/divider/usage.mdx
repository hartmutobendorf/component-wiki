The divider is a vertical line used to separate content within an interface. It creates clear boundaries between distinct sections, helping users understand the structure and organization of information.

## Visual grouping and emphasis

Dividers delineate where one group of content ends and another begins, reinforcing visual hierarchy by drawing attention to separation between discrete elements. They communicate that adjacent items, while spatially close, are not part of the same logical group. This is useful in dense interfaces where proximity alone does not sufficiently convey the relationship between elements.

Space should always be the first tool considered when separating content. Dividers are a secondary measure, appropriate only when whitespace fails to establish sufficient distinction. Overuse quickly becomes counterproductive. Too many lines create visual noise that obscures rather than clarifies structure. Additionally, avoid placing dividers adjacent to elements that already have borders, such as cards.

Determining when a divider is warranted remains a judgment call. Evaluate whether the separation genuinely improves clarity, or merely adds visual weight without benefit.

## When to Use

*   Space alone is insufficient to communicate that elements are separate
*   A clear visual boundary is needed between discrete sections
*   Reinforcing hierarchy improves usability

## When Not to Use

*   Whitespace adequately separates the content
*   Adjacent elements already have visible borders
*   Multiple dividers would appear in close proximity
*   The addition does not meaningfully improve clarity
