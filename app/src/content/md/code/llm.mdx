## Anatomy

### 1. Code

The code being displayed either in-line or standalone

## Usage

The code component displays code, commands, and technical language that requires distinct visual treatment. This includes programming languages, domain-specific languages such as search syntax, terminal commands, file paths, and configuration values. Code uses a monospace font to distinguish it from human language, which uses the default text style.

The component appears most often within paragraph text but can also be used in lists, tables, and other structured content.

## Inline code

Use inline code for short, simple references within the normal text flow, such as commands, function names, or configuration keys. Keep inline code concise. Long expressions or multi-part statements should use a code block instead.

## Code blocks

Use code blocks for code spanning more than one line or when single-line code benefits from separation from surrounding text.

### Wrapping and overflow

Code blocks can either wrap long lines or allow horizontal overflow with scrolling.

Prefer horizontal overflow when:

*   Code is meant to be copied and executed exactly as shown
*   Indentation carries syntactic meaning, such as in Python or YAML
*   Content is diagnostic, such as log output with column-based formatting

Prefer wrapping when readability matters more than preserving exact formatting, such as illustrative pseudocode or conceptual examples.

## Presenting code effectively

Introduce what the code does before showing it. After presenting code, explain what the user should notice or learn. Maintain proper indentation within code blocks.

## When to use

*   To display programming code, terminal commands, or domain-specific languages
*   For technical values that should be interpreted literally
*   To present executable snippets users may copy

## When not to use

*   For regular prose or human-readable text
*   For emphasis or stress (use bold or italics instead)
