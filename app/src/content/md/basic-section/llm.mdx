# Basic section

## Description

The basic section is a 50/50 layout pattern designed to present a single topic using various content blocks. It provides a flexible layout system that can accommodate different types of content including text, images, videos, lists, code blocks, logos, and call-to-action elements. Right side of the 50/50 layout can have one or more blocks, allowing to build clear and modular narratives without breaking the overall page rhythm.

The basic section helps create balanced, scannable content that adapts to a wide range of use cases—from simple informational layouts to more expressive, mixed-media compositions. Be mindful that its flexibility can also lead to inconsistency if too many block types or visual densities are combined. Use the basic section when the message fits within a single topic and can be communicated cleanly within a two-column structure.

## Anatomy

### 1. Rule

The rule component indicates the beginning of a new section.

### 2. Heading

The section heading communicates the primary subject or theme of the section.

### 3. Eyebrow

The optional muted eyebrow placed above the section title. It provides additional contextual information or categorization for the section when needed.

### 4. Content area

The content area on the right side of the 50/50 layout where various block types can be placed. Supported blocks include text, images, lists, videos, code snippets, logos, CTAs, and other compatible patterns. The Basic Section layout provides flexible composition while maintaining structural consistency.

## Usage

Use the Basic section when you want to communicate a focused topic supported by various types of structured content. For optimal clarity, include no more than seven content blocks within a single section. Avoid overloading the section with unrelated messages or too many mixed content types. The Basic Section works best when the narrative is concise and the supporting content is clearly connected to the section’s main topic.

The pattern is most effective when the information fits naturally into a two-column layout, with a clear title on the left and explanatory content on the right. For medium size, the layout supports both 50/50 and 100 configurations, defined as variants within the macro.

### Supported UI blocks for the content area

1.  **Subheading**: The subheading provides additional hierarchy beneath the main title, using H4 or H5.
2.  **Body**: Standard paragraph-level text used for descriptive or explanatory content.
3.  **Image**: 3:2 or 16:9 ratio image used to illustrate or enhance the section content. It can contain photography, isometric illustrations, diagrams, and similar visuals. The image can also include an optional caption.
4.  **Video**: The linked, embedded video content in 16:9 ratio.
5.  **List**: The divided list supporting up to 10 items, with bullet type options including none, bullet, status, or number.
6.  **Code block**: The pre-formatted code presentation block.
7.  **Logo block**: The logo block showcases a group of related images, such as a group of customer or partner logos.
8.  **Linked logo block**: The linked logo section displays a list of logos which explicitly link to additional pages or resources.
9.  **CTA block**: The CTA block contains one or more call-to-action elements, including a primary button, secondary button, and links.
10.  **Notification**: A prominent information block used to highlight alerts, updates, or contextual messages within the section.

### When to use

*   You need to present one clear topic with supporting explanatory content.
*   You need a flexible pattern that works for both marketing and informational content.
*   The 50/50 layout helps balance text and media for better readability.

### When not to use

*   The section contains multiple topics or subsection groups that need clear internal separation, or it must be scanned quickly in a list-like or grouped format better suited to other structural patterns.
*   *   In these cases, use a Tiered list, a Equal heights or a Divided section instead.
*   The section includes only one primary content block.
*   *   In that case, use a pattern specifically designed for that block (e.g., Logo section, Linked logo section, Resources).
*   The section requires a more immersive layout or highlights rather than detailed explanation.
*   *   In that case, use Rich vertical list or highlight-focused patterns (e.g.,Text highlight, Data highlight)
